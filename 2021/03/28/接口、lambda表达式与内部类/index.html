接口、lambda表达式与内部类

接口

java中接口不是类，是对希望符合这个接口的类的一组需求。

一个类可以实现多个或一个接口。

接口中所有方法自动为public，编写时可不写。但是实现接口的类 ，重写方法时必须声明为public，否则编译器报错

接口可以定义常量，接口没有实例字段

  • 具体实例Comparable接口

    Arrays类中的sort方法承诺可对对象数组排序、但需要满足对象所属类必须实现Comparable接口

    • Comparable接口要求

      ​ x.compareTo(y)确实能比较两个对象并返回x和y哪一个更大。

      ​ x小于y返回一个负数，x大于y返回正数，x等于y返回0.

    实现 Comparable 接口的类，都需要包含 compareTo 方法，这里 Comparable 接口为一个泛型类型，因此实现该接口的时候需要带上类型 java5

    Comparable 也可以使用不带类型参数的，但是这样在实现的方法里就有 Object 类型的参数，需要手动的进行强制转换

类实现接口步骤

  1. 类的声明实现给定接口 使用implements关键字
  2. 对接口中所有方法提供定义

例如

对于继承了实现 Comparable 接口的子类之间比较的问题

解决办法：

  • 如果不同子类中的比较有不同的含义 (意思就是同一个类中可以进行比较，但是与另一个类无法进行比较)，应该将属于不同类的对象之间的比较视为非法的。每个 compareTo 方法开始时进行检测 if(getClass() != other.getClass()) throw new ClassCastException();
  • 如果存在一个能够比较子类对象的通用算法，可以在超类中提供一个 compareTo 方法，并将这个方法声明为 final

可以声明接口变量 Comparable x;

instanceof 可用于检查一个对象是否实现了某特定接口，类似检查一个对象是否属于某个特定类

 `if (anObject instanceof Comparable){  ...  } `

回调

软件模块之间总是存在着一定的接口，从调用方式上，可以把他们分为三类：同步调用、回调和异步调用。同步调用是一种阻塞式调用，调用方要等待对方执行完毕才返回，它是一种单向调用；回调是一种双向调用模式，也就是说，被调用方在接口被调用时也会调用对方的接口；异步调用是一种类似消息或事件的机制，不过它的调用方向刚好相反，接口的服务在收到某种讯息或发生某种事件时，会主动通知客户方（即调用客户方的接口）。回调和异步调用的关系非常紧密，通常我们使用回调来实现异步消息的注册，通过异步调用来实现消息的通知。同步调用是三者当中最简单的，而回调又常常是异步调用的基础

​ 允许从通用性高的接口 到通用性低的接口有多条接口链。与建立类的继承层次一样。

​ 接口通过 extends 来实现接口的扩展

  • 接口中的方法都自动设置为 public

  • 接口中的字段总是 public static final

注：在接口中可以显示的标记出来，也可以不标记出来

  • 每个类只能有一个超类

    java中只能继承一个类 c++可以多重继承。多重继承会让语言变复杂，或效率降低（如Eiffel）

  • 每个类能够实现多个接口

    实现多个接口用逗号隔开: class Employee implements Cloneable, Comparable

java8中，允许在接口中增加静态方法

  • 目前通常做法是将静态方法放在伴随类中

默认方法

接口方法中可提供一个默认实现 使用default关键字

默认方法中可以调用其他方法

接口演化 interface evaluation

接口改变后，原有实现接口类必须实现新增方法，否则无法通过编译。通过默认方法可以避免。

默认方法的好处就是保证了源代码兼容 (source compatible)

解决默认方法冲突

情景：一个接口中有默认方法，同时在超类或者另一个接口也有定义同样的方法

java中规则如下

  1. 超类优先

    如果超类提供具体的方法，同名且有相同参数类型 (签名相同) 的默认方法会被忽略

  2. 接口冲突

    如果一个接口提供默认方法，另一个接口提供了同名且参数类型相同的方法 (无论是否是默认参数)，必须覆盖这个方法解决冲突

    • 重新写这个方法

    • 使用其中一个接口实现的方法，这里要加 super 才能够使用这个方法

      如果一个类 implements 一个接口，但是不实现里面的方法，就需要将该类定义为抽象类 (abstract)

对象克隆

  • 用 = 来为一个包含对象引用的变量建立副本时，只是将原变量和副本指向同一个对象的引用

Object 类中有一个声明为 protected 的 clone 方法

浅拷贝

  • Object 类中 clone 方法默认是浅拷贝，即没有克隆对象中引用的其他对象
  • 如果对象中的所有数字段都是数值或其他基本类型，拷贝字段没有问题
  • 如果对象包含子对象的引用，拷贝字段就会得到相同子对象的另一个引用
    • 原对象和克隆的对象会共享信息
  • 如果原对象和浅克隆对象共享的子对象是不可变的类 (例如 String)，则共享安全
  • 如果共享子对象一直包含不变的常量、没有更改其方法改变它、没有方法生成引用，则也是安全

深拷贝

  • 克隆对象的时候同时克隆所有子对象

对于克隆对象需要确定几点

  1. 默认的 clone 方法是否满足要求（通常子对象可变，必须重新定义clone实现深拷贝）
  2. 是否可以在可变的子对象调用 clone 来修补默认的 clone 方法
  3. 是否不该使用 clone

如果需要使用 clone 方法有以下步骤

  1. 实现 Cloneable 接口
  2. 重新定义 clone 方法，并指定 public 访问修饰符

Cloneable 里面没有方法，clone 方法是从 Object 类继承的

只是一个标记接口，如果一个对象没有实现这个接口但请求克隆，会生成一个检查型异常

标记接口 (tagging interface) 记号接口 (marker interface)

  • 标记接口不包含任何方法
  • 唯一作用是允许在类型查询中使用 instanceof
    • 例如 if(obj instanceof Cloneable)...

方法引用

lambda表达式会涉及一个方法

表达式 System.out::println 是一个方法引用，它指示编译器生成一个函数式接口的实例，覆盖这个接口的抽象方法来调用给定的方法

使用方法引用可改写成

类似于lambda表达式，方法引用也不是一个对象，但是为一个类型为函数式接口的变量赋值时，会生成一个对象。

对字符串排序，忽略大小写

用:: 运算符分隔方法名与对象或类名

  1. object::instanceMethod
  2. Class::instanceMethod
  3. Class::staticMethod

  • 第一种情况，方法引用等价于向方法传递参数的 lambda 表达式:System.out::println — x->System.out.println(x)//没懂，前一个 打印空行

  • 第二种情况，第一个参数变成方法的隐式参数：Arrays.sort(planets,String::compareToIgnoreCase); — (x,y)->x.compareToIgnoreCase(y)

  • 第三种情况，所有参数都传递到静态方法:Math::pow — (x,y)->Math.pow(x,y)

注意

只有当lambda表达式的体只调用一个方法而不做其他操作时，才能把lambda表达式重写为方法引用。

方法引用和 lambda 表达式区别之一在于：lambda 只有在调用的时候，才会抛出异常。方法引用在构造的时候就会抛出异常

​ 如 separator::equals。如果separator为null。separator::equals在构造时会抛出NullPointException异常。lambda表达x->separator.equals(x)在调用时才会抛出异常

方法引用中可以使用 this 参数，例如 this::equals 等同于 x -> this.equals(x)

方法引用使用 super 参数， super::instanceMethod

内部类

内部类是一个定义在另一个类中的类

使用原因

  • 内部类可以对同一个包中其他类隐藏
  • 内部类可以访问定义这个类的作用域中的数据，包括私有的数据

内部类访问对象状态

一个内部类方法可以访问自身的数据字段，也可以访问创建它的外围类对象的数据字段

内部类位于外部类的内部，不代表每个外部类都有一个内部类的实例字段。

内部类的对象总有一个隐式引用，指向创建它的外部类的对象。

  • 这个引用在内部类的定义是不可见的 (即没有显示的定义这个隐式引用)

  • 外围类的引用在内部类的构造器中设置

    编译器会修改所有内部类的构造器，添加一个对应外围类引用的参数

内部类特殊语法规则

外围类引用的正规语法

outerObject.new InnerClass(construction parameters)

  • 内部类所有声明的静态字段必须是final，并初始化为一个编译时常量。

    ​ 如果这个字段不是一个常量，就可能不唯一。

  • 内部类不能有static方法

内部类是一个编译器现象，与虚拟机无关。编译器会把内部类转换成常规的类文件，用 $ 分割外部类名与内部类名，虚拟机对此一无所知

例如 OuterClass$InnerClass

编译器会额外生成一个实例字段this$0（编译器合成，自己编写代码中不能引用）,对应外围类引用。

与直接写一个常规类来访问某一个类相比，内部类拥有更大的访问权限。内部类可以访问外部类的私有数据，而常规类不能访问。

局部内部类

即在一个方法中局部的定义这个类

  • 当内部类在外部类中名字只出现一次的时候 (只是在这个方法中创建这个类型的对象时使用了一次)，可以将内部类改写成局部内部类

  • 声明局部类不能有访问说明符 （即publi或private）

    ​ 局部类的作用域被限定在声明这个局部类的块中

  • 局部类对外界完全隐藏

    ​ 外围类其他方法也不能访问，除定义它的方法外，没有任何方法知道局部类的存在

静态内部类

静态内部类：将内部类声明为 static 就不会生成那个外围类引用

目的：只是为了把一个类隐藏在另一个类的内部，并不需要内部类有外围对象的一个引用。

规范：质只要内部类不访问外围类对象，就应该使用静态内部类

只有内部类可以声明为 static

当静态方法中构造一个内部类，则该内部类必须是静态的

  • 与常规内部类不同，静态内部类可以有静态字段和方法
  • 接口中声明的内部类自动是 static 和 public
